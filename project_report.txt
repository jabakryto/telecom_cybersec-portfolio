1. ВВЕДЕНИЕ
-----------
Цель данной работы — практическое освоение методов поиска и эксплуатации
уязвимостей в веб-приложениях и сетевых сервисах с использованием Kali Linux.
Использованы специально уязвимые системы:
- DVWA (Damn Vulnerable Web Application) — для тестирования веб-уязвимостей.
- Metasploitable 2 — для анализа уязвимостей сетевых сервисов.

Все результаты зафиксированы в рамках подготовки к специализации
в области кибербезопасности телекоммуникационных систем.

2. АНАЛИЗ DVWA (http://127.0.x.x/dvwa/)
-----------------------------------------

2.1 SQL Injection (уровень Low)
Payload: ' OR '1'='1
Результат: Получен полный список пользователей БД (admin, gordonb, 1337 и др.).
Вывод: Приложение не использует параметризованные запросы → уязвимость подтверждена.

2.2 Reflected XSS
Payload: <script>alert('XSS')</script>
Результат: Выполнено всплывающее окно в браузере.
Вывод: Входные данные не экранируются при выводе → возможна кража сессий.

2.3 Использование Burp Suite
- Настроен прокси (127.0.x.x:8080).
- Перехвачен POST-запрос при входе в систему.
- Модифицирован параметр username на ' OR '1'='1 -- → успешный вход без пароля.
- Подтверждена возможность ручной эксплуатации уязвимостей через перехват трафика.

3. РЕЗУЛЬТАТЫ АВТОМАТИЗИРОВАННОГО СКАНИРОВАНИЯ DVWA
--------------------------------------------------

3.1 Nikto (nikto -h http://127.0.x.x/dvwa/)
Файл: nikto_dvwa.txt
Ключевые находки:
- Отсутствуют заголовки безопасности: X-Frame-Options, X-Content-Type-Options.
- Обнаружена страница /dvwa/login.php — стандартная точка входа.
- Сервер: Apache/2.4.52 (Ubuntu) — потенциально уязвим при отсутствии патчей.

Рекомендации:
- Добавить заголовки безопасности.
- Ограничить доступ к /dvwa/ извне.
- Обновить веб-сервер и PHP.

3.2 Dirb (dirb http://127.0.x.x/dvwa/ /usr/share/dirb/wordlists/common.txt)
Файл: dirb_dvwa.txt
Найдены пути:
- /dvwa/admin/ (302 Found)
- /dvwa/vulnerabilities/ (302 Found)
- /dvwa/logout.php (302 Found)

Вывод: Структура приложения открыта → злоумышленник может изучить логику работы.

4. СКАНИРОВАНИЕ METASPLOITABLE
-------------------------------

Цель: [IP из metasploitable_ip.txt, например: 192.168.1.105]
Файл с результатами: metasploitable_scan.txt

Команда: nmap -sV -sC -p- 192.168.1.105

Ключевые уязвимые сервисы:
- FTP (vsftpd 2.3.4) — известная backdoor-уязвимость.
- SSH (OpenSSH 4.7p1) — устаревшая версия, возможен brute-force.
- Telnet — незашифрованный протокол, передача данных в открытом виде.
- Samba (3.0.20-Debian) — уязвимость в аутентификации (CVE-2007-2447).
- MySQL, PostgreSQL — открыты порты, возможен перебор учётных данных.

Вывод: Система содержит множество критических уязвимостей,
позволяющих получить полный контроль (RCE, LFI, brute-force).

5. ЗАКЛЮЧЕНИЕ И РЕКОМЕНДАЦИИ
-----------------------------

Работа подтвердила:
- Критическую важность валидации входных данных в веб-приложениях.
- Необходимость регулярного сканирования инфраструктуры на уязвимости.
- Риски использования устаревшего ПО (как в DVWA, так и в Metasploitable).

Рекомендации по повышению безопасности:
1. Для веб-приложений:
   - Использовать параметризованные запросы (SQLi).
   - Экранировать вывод (XSS).
   - Настроить CSP и безопасные HTTP-заголовки.
2. Для сетевой инфраструктуры:
   - Отключить ненужные сервисы (Telnet, FTP).
   - Обновлять ПО до актуальных версий.
   - Использовать брандмауэры и IDS/IPS.
3. Для телеком-сред:
   - Применять шифрование сигнального трафика (например, для Diameter over TLS).
   - Мониторить аномалии в GTP/Diameter-трафике.

Данный отчёт может быть использован как основа для дальнейшего изучения
безопасности телекоммуникационных протоколов (SS7, GTP, IMS) и участия
в CTF-соревнованиях и пентест-проектах.
